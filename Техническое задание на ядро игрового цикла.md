## **2. Функция проверки блокировок **`check_day_blocked()`

Блокирует день, если:

- Есть необработанное событие (`game_state.pending_event != null`).

- Пустой склад (для бизнесов со складом) и нет возможности закупить (денег нет или закупка не предложена). Но точнее: если `stock == 0` и игрок не совершил закупку (флаг `stock_emergency_block`).

Псевдокод:

```
function check_day_blocked():
    if game_state.pending_event:
        show_message("Сначала решите событие.")
        return false
```

```

        else:
            # Игрок отказался закупить (нажал "Отложить"), но склад пуст - блокируем
            show_message("Склад пуст. Сделайте закупку.")
            return false
```

## **5. FIFO реализация (псевдокод)**

Склад хранится как список партий:

```
struct Batch:
    id: int
    quantity: int
    price_per_unit: float
    day_received: int
    expiration_days: int
```

Функции:

```
function fifo_consume(units):
    remaining = units
    total_cost = 0
    while remaining > 0 and game_state.batches:
        batch = game_state.batches[0]
        take = min(batch.quantity, remaining)
        batch.quantity -= take
        remaining -= take
        total_cost += take * batch.price_per_unit
        if batch.quantity == 0:
            game_state.batches.pop(0)
    # Если remaining > 0, значит товара не хватило
    served_units = units - remaining
    return served_units, total_cost
function fifo_check_expired():
    today = game_state.day
    total_loss = 0
    for batch in game_state.batches:
        age = today - batch.day_received
        if age >= batch.expiration_days:
            loss_multiplier = 0.8  # 80% убыток
            if game_state.services.Market:
                loss_multiplier = 0.64  # 64%
            total_loss += batch.quantity * batch.price_per_unit * loss_multiplier
            # Удаляем просроченную партию
            batch.quantity = 0
    # Удалить пустые партии
    game_state.batches = [b for b in game_state.batches if b.quantity > 0]
    return total_loss
```

## **6. Генерация событий **`generate_events()`

Возвращает список событий, которые должны произойти в этом дне (обычно 0 или 1).

Алгоритм:

- Пройти по всем событиям из библиотеки.

- Проверить триггеры (день, репутация, бизнес, random).

- Для случайных событий: шанс `random:0.05` означает 5% вероятность, если остальные условия выполнены.

- Собрать все подходящие события.

- Если их несколько, выбрать одно случайное (или приоритет по важности).

- Вернуть выбранное событие (или пустой список).

```
function generate_events():
    candidates = []
    for event in events_library:
        if check_event_trigger(event, game_state):
            candidates.append(event)
    if not candidates:
        return []
    # Случайный выбор
    chosen = random.choice(candidates)
    # Событие может иметь несколько вариантов, но они выбираются игроком позже
    return [chosen]
```
