### 3. Обновление App.tsx — управление экранами
**Логика:**
```tsx
// 1. На старте загружаем состояние из localStorage
useEffect(() => {
  const saved = localStorage.getItem('konturgame_state')
  if (saved) loadGame(JSON.parse(saved))
  setLoaded(true)
}, [loadGame])

// 2. Если нет сохранений → BusinessSelector
if (!hasStartedGame) {
  return <BusinessSelector />
}

// 3. Иначе → MainScreen (основная игра)
return <MainScreen />
```

**Преимущества:**
- Чистое разделение ответственности
- Легко расширить (например, экран меню)
- Состояние синхронизировано с localStorage
- Плавный переход между экранами

### 4. Обновление SettingsModal
**Изменение:** Кнопка "Начать новую игру"
```tsx
// Было:
startNewGame('shop')

// Стало:
localStorage.removeItem('konturgame_state')
window.location.reload()
// → Показывается BusinessSelector
```

**Результат:** Пользователь может в любой момент вернуться на экран выбора типа бизнеса

## 🚀 Как это работает

### Первый запуск
1. Пользователь открывает приложение
2. App.tsx проверяет localStorage
3. Сохранений нет → показывается **BusinessSelector**
4. Пользователь выбирает тип бизнеса
5. Вызывается `startNewGame(businessType)`
6. Состояние сохраняется в localStorage
7. Приложение перерендеривается и показывает **MainScreen**

### Последующие запуски
1. Пользователь открывает приложение
2. App.tsx загружает состояние из localStorage
3. Показывается **MainScreen** с сохраненным прогрессом

### Новая игра
1. Пользователь нажимает "Начать новую игру" в Настройках
2. localStorage очищается
3. Страница перезагружается
4. App.tsx показывает **BusinessSelector** снова
