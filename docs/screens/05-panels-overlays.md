# 05. Панели, оверлеи, дизайн-система

Компоненты, которые не являются ни модалками, ни полноценными экранами, но встроены в макет. Плюс — обзор дизайн-системы (`src/components/design-system/`).

## Panels

### OnboardingPanel

**Файл:** `src/components/OnboardingPanel.tsx`.

**Роль:** коллапсируемая панель с текущим шагом онбординга. На десктопе — над контентом в `MainScreen`, на мобильном — на вкладке «Сервисы».

**Что внутри:**

- Заголовок текущей стадии (Stage 0–4) с цветовой подсветкой: синий → фиолетовый → оранжевый → мятный → оранжевый.
- Индикатор прогресса (5 точек).
- Номер текущего шага и общее количество шагов в стадии.
- Тело шага:
  - **intro** — текст + кнопка «Понял →».
  - **action** — оранжевый фон «требуется действие», кнопка-CTA со стрелкой, переводящая на нужную вкладку (например, «Открыть Finance»).
  - **wait** — фиолетовый фон, пульсирующая точка, подсказка «Что сделать, чтобы продолжить».
- Кнопка сворачивания.

**Логика:** читает `onboardingStage`, `onboardingStepIndex` из стора; вызывает `advanceOnboarding()` после нажатия кнопки.

### ServicePanel

**Файл:** `src/components/ServicePanel.tsx`.

**Роль:** компактная панель управления сервисами Контура (альтернатива `DesktopKontur` для небольших экранов или правой колонки).

**Состояния:**

- **Свёрнута:** кнопка «📘 Сервисы Контура», 4×N сетка иконок с зелёными точками у активных, счётчик синергий.
- **Развёрнута:**
  - список «Доступные» — каждый сервис: иконка, название, цена/год, описание, toggle;
  - список «Закрытые» — серый фон с замком, подписью «Откроется на этапе X»;
  - блок «Активные синергии» с коротким описанием каждой.

### Indicators

**Файл:** `src/components/Indicators.tsx`.

**Роль:** панель из 7 прогресс-баров, основная мобильная вкладка «Индикаторы».

**Индикаторы:**

1. **⭐ Репутация (0–100)** — статус mint / orange / red.
2. **❤️ Лояльность (0–100)** — с кнопкой «💰 Премия персоналу» (видна если лояльность < 70, стоимость = `revenue × 0.2`, добавляет лояльность).
3. **⚡ Энергия/выгорание (0–100)** — восстанавливается в начале каждой недели.
4. **📦 Склад (%)** — заполнение, если у бизнеса есть склад.
5. **✨ Качество (0–100)** — показатель текущего `qualityLevel`.
6. **🏢 Стадия бизнеса** — текстом, с условиями следующей стадии.
7. **👥 Обслуженность дня** — разбитый бар: зелёное (обслужено) + красное (упущено).

### NextDayButton

**Файл:** `src/components/NextDayButton.tsx`.

**Роль:** главный CTA игры — завершить день/неделю.

**Состояния:**

- **Normal** — оранжевый, «Завершить неделю →».
- **Event Pending** — светлый, заблокирован, «⏸ Разрешите событие».
- **Loading** — «⏳ Расчёт…», заблокирован во время `processWeek()`.
- **Game Over** — «❌ Игра окончена», заблокирован.
- **Victory** — «🎉 Победа!», заблокирован.

Проверяет `pendingEvent` перед действием. При попытке нажать с активным событием — тост-предупреждение.

## Overlays

### WeekSummaryOverlay

**Файл:** `src/components/WeekSummaryOverlay.tsx`.

**Роль:** приветственный оверлей в начале новой недели (`weekPhase === 'summary'`).

**Содержимое:**

- Заголовок «Неделя N».
- Рекап прошлой недели (выручка, чистая прибыль, баланс).
- Панель «Энергия владельца» с прогресс-баром и текстом «Энергия восстановлена. Планируйте неделю».
- Счётчик подключённых сервисов (N из 7) с подсказкой.
- CTA «Начать неделю →» → `startWeekActionsPhase()`.

### WeekResultsOverlay

**Файл:** `src/components/WeekResultsOverlay.tsx`.

**Роль:** финальный оверлей недели (`weekPhase === 'results'`).

**Содержимое:**

1. **Блок вехи** (если достигнута week10/20/30) — градиент violet→blue, поздравление.
2. **Чистая прибыль** крупным шрифтом (чёрный если плюс, красный если минус); рядом — текущий баланс.
3. **Финансовый разбор:** выручка, закупки, налог, подписки Контура, списание просрочки, штраф за перегруз касс, итого.
4. **Статистика:** клиентов обслужено из X, сколько сервисов Контура активны, сколько достижений получено.
5. **Блок Pain Loss (недели 1–6):** «Без сервисов Контура потеряно: X ₽», топ-3 источника потерь.
6. **Постоянный клиент** — если он есть: зелёная карточка «Был у вас на этой неделе» или оранжевая «Не заходил — теряем лояльность».
7. **Тизер следующей недели** (опционально) — подсказка о грядущем событии.
8. CTA «Следующая неделя →».

## Design System

Папка `src/components/design-system/` содержит переиспользуемые блоки макета и токены.

### tokens.ts

Палитра `K`:

```
paper / bone / ink / ink2 / muted / line   — нейтральные (фоны, текст, границы)
mint (+ mintSoft, mintInk)    — success, сервисы Маркет/Диадок/Экстерн
blue (+ blueSoft, blueInk)    — Банк, Эльба
violet (+ violetSoft, violetInk) — ОФД, события, фиолетовая фаза
orange (+ orangeSoft)         — alerts, Фокус, рекламные акценты
good / warn / bad             — семантические цвета статусов
```

Все компоненты используют только эти значения — единая визуальная тема.

### KLeftRail

Вертикальная панель слева в `MainScreen`.

- Наверху — иконка типа бизнеса, название, неделя, сезон.
- Восемь вкладок: dashboard / ecosystem / finance / warehouse / operations / development / statistics / journal. Каждая с SVG-иконкой, активная подсвечена, заблокированные серые с замком.
- Снизу — утилитарные кнопки: ❓ Help, ⚙️ Settings, 🏆 Achievements (с бейджем), 🎟️ Promo Wallet.

### KHeaderBar

Верхняя строка статуса.

- Слева: тип бизнеса, «Неделя N · День X из 7», сезон.
- Справа: бейдж фазы недели с цветом:
  - `summary` → blue «Итоги прошлой недели»;
  - `actions` → violet «Управляйте бизнесом»;
  - `events` → violet «Требуется решение»;
  - `results` → mint «Неделя завершается».

### DesktopKontur

Полноэкранная витрина сервисов (вкладка `ecosystem`).

- Слева — 2×3 сетка из 7 сервисов (в двух колонках). Активный сервис — цветной фон (mint/blue/violet/orange), неактивный — белая карточка с обводкой. Клик — toggle.
- Справа — информационная панель: список активных сервисов, вычисленная «экономия» (сумма предотвращённых pain-losses), активные синергии.
- При активации сервиса — всплывает toast с описанием эффекта.

### DesktopDashboard

Сборка дашборда: KPI-карточки, чек-лист, виджеты очереди и склада.

### DesktopEvent

Карточка события: тёмный фон, портрет NPC, текст, варианты с тегами. Используется в Dashboard.

### KStatusBar

Компонент-пилл со статусом (репутация/лояльность/энергия) — маленький пиллообразный виджет с цветом и числом. Используется в правой панели и в оверлеях.

### primitives.tsx

`Row`, `Col`, `Card` — простые layout-компоненты с консистентными отступами.
