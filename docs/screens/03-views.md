# 03. Вкладки (Views) в MainScreen

Все эти компоненты лежат в `src/components/views/` (кроме Dashboard, собранной из дизайн-системы в `MainScreen`). Игрок переключается между ними через `KLeftRail`.

## Dashboard

**Источник:** собирается прямо в `MainScreen.tsx` из компонентов дизайн-системы (`DesktopDashboard`, `DesktopEvent`, `KStatusBar`).

**Роль:** главный экран управления в текущем дне. Всё, что нужно делать регулярно, — здесь.

**Что показывает:**

- 4 KPI-карточки: баланс, прибыль за прошлый день, расходы, обслужено клиентов.
- Чек-лист «что нужно сделать сегодня» (например, решить событие).
- Карточка текущего события (если есть) с вариантами ответа и тегами «Контур» / «Риск».
- Виджет очереди (обслужено vs упущено).
- Виджет склада (FIFO-партии с днями до просрочки).
- Правая панель: статус-пилы, синергии, стадия, сервисы 2×3, кнопка «Следующий день».

## FinanceView

**Файл:** `src/components/views/FinanceView.tsx`.

**Роль:** финансовый отчёт и управление кредитами.

**Секции:**

1. **Баланс к цели** — прогресс-бар до 500 000 ₽ и процент.
2. **Доход за вчера** — выручка.
3. **Расходы (детализация)** — закупки, налог УСН 6%, аренда и зарплата (плановый платёж), подписки Контура, списание просрочки, штраф за переполнение очереди.
4. **Боль-потери (без сервисов)** — список того, что теряется из-за каждого неактивного сервиса (см. 03-services-and-pain.md).
5. **Месячные расходы** — аренда, базовая зарплата, зарплата сотрудников, подписки (сумма `annualPrice / 12`), итого.
6. **Прогноз** — балансовая проекция на 4 и 8 недель на основе последней недельной прибыли.
7. **Кредиты** — список активных кредитов с кнопкой «Погасить»; форма взятия нового с 3 вариантами (микрозайм / стандартный / долгосрочный). Кнопки доступны с учётом наличия Контур.Банка.

## WarehouseView

**Файл:** `src/components/views/WarehouseView.tsx`.

**Роль:** обзор товарного ассортимента. Для салона красоты показывает категории услуг.

**Структура:**

- Заголовок и кнопка «🛍️ Управлять ассортиментом» → `AssortmentModal`.
- 3 KPI-карточки: количество активных категорий, суточная стоимость закупок, базовая выручка в день.
- Список активных категорий (иконка, название, описание, требуемые сервисы).
- Если активных нет — пустое состояние с CTA.

Для бизнесов без склада (`hasStock: false`) показывает только услуги.

## OperationsView

**Файл:** `src/components/views/OperationsView.tsx`.

**Роль:** операционное управление — персонал + ассортимент + стадия бизнеса.

**Секции:**

1. **Ассортимент (если есть)** — список категорий с toggle. Для каждой: иконка, название, требуемые сервисы, статус (✓ зелёный — все сервисы есть, ! оранжевый — нет), подсказка о недостающих.
2. **Сотрудники** — счётчик `N / max` (макс зависит от `businessStages.ts`), список нанятых с должностью, зарплатой и кнопкой «Уволить», кнопка «+ Нанять» открывает `HireEmployeeModal`.
3. **Стадия бизнеса** — текущая стадия с описанием эффектов; условия разблокировки следующей.

## DevelopmentView

**Файл:** `src/components/views/DevelopmentView.tsx`.

**Роль:** всё, что связано с ростом — реклама, улучшения, аналитика кампаний.

**Три вкладки внутри:**

1. **Реклама** — слоты под одновременные кампании (визуализация квадратиков). Список доступных кампаний: название, стоимость, длительность, эффект на клиентов, кнопка «Запустить». Показатель убывающей отдачи: «следующий слот даст X%».
2. **Улучшения** — список апгрейдов для текущего типа бизнеса с иконкой, описанием эффекта, стоимостью. Купленные отмечены галочкой. Альтернативный UI — `UpgradesModal`.
3. **ROI кампаний** — таблица истории запущенных кампаний: название, цена, прирост клиентов, возврат инвестиций, неделя.

## StatisticsView

**Файл:** `src/components/views/StatisticsView.tsx`.

**Роль:** агрегированная статистика текущего забега и история всех забегов.

**Секции:**

1. **Текущий забег — прогресс:**
   - сыграно недель, прогресс-бар к 52;
   - баланс, репутация, лояльность, уровень;
   - активных сервисов / 7;
   - вехи достигнуты / 3;
   - запущенных кампаний, ачивок / всего.
2. **История забегов** — читается из localStorage (метастора): всего игр, выиграно, средняя длина, лучший баланс, таблица последних 10 забегов.
3. **Кнопка «📤 Поделиться»** — Web Share API или копирование в буфер обмена с текстом об итогах забега.

## DecisionLogView

**Файл:** `src/components/views/DecisionLogView.tsx`.

**Роль:** хронологический журнал всех принятых решений и сюжетных событий. Разблокируется с 10-й недели.

**Что показывает:** список `decisionLog` в обратном порядке. Каждая запись:

- бейдж номера недели;
- тип: «Решение», «Цепочка», «Веха», «NPC», «Газета», «Клиент»;
- иконка влияния (↑ позитив, ↓ негатив, — нейтрально);
- текст решения и последствия;
- ID NPC, если запись связана с персонажем.

Записи имеют цветовую кодировку: зелёный / красный / серый, левая граница карточки покрашена в цвет типа.

## Разблокировка вкладок

Определяется в `src/constants/onboarding.ts` (`VIEW_UNLOCK_MAP`):

| Вкладка | Доступна с |
|---------|------------|
| `dashboard`, `ecosystem`, `finance`, `operations` | с 1-го дня |
| `warehouse` | неделя 2 |
| `development` | неделя 2 |
| `statistics` | неделя 7 |
| `journal` | неделя 10 |

При первой разблокировке в `MainScreen` всплывает тост.
