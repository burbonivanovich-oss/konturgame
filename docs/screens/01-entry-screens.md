# 01. Экраны входа

Экраны, которые видит игрок до того, как попадает в основной MainScreen.

## ResponsiveLayout

**Файл:** `src/components/ResponsiveLayout.tsx`.

**Роль:** Метакомпонент-переключатель. Определяет по ширине окна, показать ли `MainScreen` (desktop) или `MobileMainScreen` (мобильный).

**Как работает:**

- Слушает событие `resize`, хранит `window.innerWidth` в локальном state.
- Порог — `1024px`. Меньше — mobile, больше — desktop.
- На мобиле оборачивает `MobileMainScreen` в рамку «Phone» (визуально изображается корпус смартфона).

**Состояние из стора:** не читает напрямую — только передаёт пропсы.

**Действия пользователя:** нет. Автоматическая реакция на изменение размера окна.

## BusinessSelector

**Файл:** `src/components/BusinessSelector.tsx`.

**Роль:** Первый экран новой игры — выбор типа бизнеса.

**Что показывает:** 3 большие карточки.

- 🏪 **Магазин** (`shop`). Клиентов мало, но чек стабильный; есть склад на 10 дней; сезон — лето +5%.
- ☕ **Кафе** (`cafe`). Много клиентов, низкий чек; скоропортящийся склад на 7 дней; сильная сезонность: лето +22%, зима −15%.
- 💅 **Салон красоты** (`beauty-salon`). Клиентов мало, но чек 400 ₽; склада нет (услуги); весна +12%.

Под каждой карточкой — табличка стартовых параметров (клиенты/день, средний чек, вместимость, аренда, зарплата, склад) и CTA «Начать с этого».

**Действия:**

- Клик по карточке → `startNewGame(businessType)` в `gameStore.ts` → переход на `BackstoryScreen`.

**UI-элементы:** крупные иконки-эмодзи, таблицы 2×3, hover-эффект (карточка поднимается).

## BackstoryScreen

**Файл:** `src/components/BackstoryScreen.tsx`.

**Роль:** Кастомизация персонажа. Выбор даёт небольшие стартовые бонусы и, главное, ветвит реплики и тост-подсказки первых недель.

**Структура:** два вопроса, каждый с 3 вариантами.

### Вопрос 1 — «Как вы здесь оказались?»

| Вариант | ID | Эффект |
|---------|-----|--------|
| 🏢 Надоела корпорация | `corp` | +10 стартовой репутации |
| 🏆 Выиграл грант | `contest` | полный начальный капитал, хорошее настроение |
| 🎲 Так получилось | `accident` | +15 стартовой энергии |

### Вопрос 2 — «Кто рядом?»

| Вариант | ID | Эффект |
|---------|-----|--------|
| 🧭 Полная свобода | `free` | нейтрально |
| 🤝 Лучший друг — болельщик №1 | `friend` | мелкий бонус репутации, особая реплика |
| 🏘️ Возвращение в свой район | `hometown` | особые реплики об NPC |

**Действия:**

- Выбор мотивации и выбор персонального контекста.
- Кнопка «Выбрать тип бизнеса →» становится активной только когда оба ответа выбраны.
- Результат уходит в `gameStore.startNewGame(...)` как `backstory: { motivation, personal }`.

## PerkSelectionScreen

**Файл:** `src/components/PerkSelectionScreen.tsx`.

**Роль:** Мета-экран между забегами. Появляется после проигрыша (не в первом забеге). Игрок выбирает один постоянный перк для следующего старта.

**Стейт:** `src/stores/metaStore.ts` (`useMetaStore`).

- `totalRuns` — количество проигрышей/стартов.
- `getAvailablePerks()` — какие перки разблокированы (открываются по `totalRuns`).
- `selectedPerk` — текущий выбор (или `null`, если решил стартовать без перка).

### Примеры перков

| ID | Эффект |
|----|--------|
| `extra_capital` | +20 000 ₽ стартового баланса |
| `rent_grace_week1` | первая неделя без аренды |
| `bank_headstart` | Контур.Банк активен с 1-го дня |
| `reputation_boost` | +15 к стартовой репутации |
| `energy_boost` | +20 к стартовой энергии |

**Действия:**

- Клик по перку → toggle (выбрать/снять).
- Кнопка «Начать с преимуществом →» (если выбран) или «Пропустить и начать →» (если нет).
- Применение перка — в `startNewGame()`: соответствующие поля стора инициализируются с бонусом.

**Связь:** после `startNewGame` игрок сразу попадает в `MainScreen`.
