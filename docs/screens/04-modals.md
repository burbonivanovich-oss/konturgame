# 04. Модальные окна

Все модалки лежат в `src/components/modals/`. У них общий props-контракт:

```ts
interface ModalProps {
  isOpen: boolean
  onClose: () => void
  // остальные пропсы специфичны для модалки
}
```

## Modal (базовый)

**Файл:** `Modal.tsx`.

Базовый компонент-обёртка: затемнение фона (overlay), контейнер с max-width, кнопка закрытия, опциональные header/footer со слотами. Все остальные модалки построены на нём.

---

## AchievementsModal

Показывает все 20 достижений.

- Разбиты на 4 волны (см. 07-progression.md в механиках).
- Заблокированные — серый фон, иконка-замок.
- Разблокированные — зелёный фон, иконка трофея.
- Прогрессивные (например, «10 дней без просрочки») — прогресс-бары.

## AssortmentModal

Управление активными категориями товаров / услуг.

- Список категорий по типу бизнеса (`PRODUCT_CATEGORIES[businessType]`).
- Каждая: иконка, название, описание, требуемые сервисы, toggle.
- Фильтр «доступные / недоступные».
- Счётчик «активных N из M».
- Кнопка «Применить» — сохраняет `enabledCategories` в сторе.

## BundleModal

Промо-бандлы: комбо сервисов со скидкой, появляются как специальные предложения.

- Название бандла, список сервисов, обычная цена vs. цена бандла, экономия %.
- Кнопка «Активировать» — оплачивает и включает несколько сервисов сразу.

## CampaignModal

Запуск рекламной кампании (альтернатива / дополнение к вкладке «Реклама» в `DevelopmentView`).

- Список кампаний: название, стоимость, длительность, эффект на клиентов/чек.
- Кнопка «Запустить» с проверкой: достаточно ли денег, есть ли свободный слот (лимит зависит от стадии бизнеса).
- Показывает убывающую отдачу (diminishing returns).

## CashRegisterModal

Покупка и управление кассами.

- Список уже купленных касс: тип, пропускная способность, состояние (рабочая / сломана).
- Возможность купить новую кассу: 3 типа (Mobile / Reliable / Fast).
- Скидки на вторую и третью (−10% / −15%).
- Подсказка: «текущая пропускная способность» и «рекомендация для вашего трафика».

## EventModal

Аналог карточки события на мобильном устройстве.

- Заголовок, описание события.
- Портрет NPC (если есть).
- Варианты ответов с иконками (Контур / Риск / Нейтрально).
- Счётчик «событие 1 / 3», если в очереди несколько.

## HelpModal

Справка по игре.

- Разделы: «Как играть», «Сервисы Контура», «Мета-прогрессия», «Контакты».
- Каждый с описанием и примерами.
- Ссылки на документацию проекта.

## HireEmployeeModal

Найм сотрудника.

- Выбор должности (6 позиций из `employees.ts`).
- Отображение зарплаты, диапазона эффективности, стоимости энергии.
- Проверки: достаточно ли баланса на первый месяц, достаточно ли энергии, не превышен ли лимит стадии.
- Кнопка «Нанять».

## MicroEventModal

Мелкие ежедневные события (без развилок).

- Заголовок, текст, иконка.
- Одно изменение показателя (например, «+1 лояльность»).
- Кнопка «Хорошо».

Появляется отдельно от «больших» событий, чтобы не перекрывать их.

## OwnerInvestmentsModal

Инвестиции владельца в себя — восстановление энергии за деньги.

- Текущая энергия (прогресс-бар).
- Варианты:
  - +10 энергии за 10 000 ₽;
  - +20 энергии за 18 000 ₽;
  - +50 энергии за 40 000 ₽.
- Объяснение, как энергия влияет на игру (падение до 0 = выгорание = game over).

## PromoCodeModal

Ввод промо-кода вручную.

- Input для ввода, кнопка «Применить».
- История использованных кодов.
- При успехе — показывает награду (деньги / сервис / бандл).

## PromoWalletModal

Кошелёк найденных промо-кодов.

- Счётчик «доступных кодов: N».
- Список всех обнаруженных кодов с описанием награды и статусом (использован / нет).
- Кнопка «Использовать» на каждом неиспользованном.

## SettingsModal

Параметры игры.

- Громкость, сложность, язык (если поддерживается).
- Очистка истории забегов.
- Кнопка «**Начать заново**» — сбрасывает `gameStore`, возвращает на `BusinessSelector`.
- Версия игры, ссылка на исходники.

## UpgradesModal

Подробное управление улучшениями (альтернатива вкладке «Улучшения» в `DevelopmentView`).

- Фильтр по категории (помещение / оборудование / маркетинг / персонал).
- Каждый апгрейд: иконка, название, описание эффекта, стоимость, требуемый уровень/стадия.
- Статусы: куплено (✓) / доступно / заблокировано.
- Кнопка «Купить».

## VictoryModal

Экран окончания забега.

**При победе:**

- 🎉 Большой заголовок «ПОБЕДА!».
- Тип победы: «Пережили первый год», «Миллион на счету», «Все сервисы подключены» и т. д.
- Финальные показатели: баланс, репутация, уровень, количество достижений.
- Кнопка «Начать заново» (с переходом на `BusinessSelector` или `PerkSelectionScreen`).

**При поражении (`defeat`):**

- 💔 Заголовок «БАНКРОТСТВО» / «ВЫГОРАНИЕ» / «ПОТЕРЯ РЕПУТАЦИИ» (по `gameOverReason`).
- Сколько недель прожили, финальный баланс, причина.
- Подсказка про выбор перка в следующем забеге.
- Кнопка «Попробовать снова» → `PerkSelectionScreen`.

## Как модалки открываются

Все модалки управляются локальным state в `MainScreen.tsx` (и соответствующих вьюхах):

```ts
const [isAchievementsOpen, setIsAchievementsOpen] = useState(false)
// ...
<AchievementsModal isOpen={isAchievementsOpen} onClose={() => setIsAchievementsOpen(false)} />
```

Открываются из:

- кнопок в `KLeftRail` (нижние утилитарные иконки) — Help, Settings, Achievements, Promo Wallet;
- быстрых кнопок мобильного экрана («Закупка», «Касса», «Энергия»);
- CTA внутри вьюх (например, «+ Нанять» в `OperationsView`, «Управлять ассортиментом» в `WarehouseView`).

Автоматически показываются:

- `MicroEventModal` — при генерации микрособытия;
- `VictoryModal` — при `isVictory || isGameOver`.
