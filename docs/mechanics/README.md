# Механики игры «Бизнес с Контуром»

Документация описывает то, что реально реализовано в коде (`src/`). Документы в корне репозитория (`GAME_MECHANICS.md`, `GAME_DESIGN_DOCUMENT.md` и др.) — это дизайн-концепты, часть из которых ещё не в коде. Единственным источником правды для чисел всегда служит `src/constants/business.ts`.

## Как читать эти документы

- Для каждой механики указано: **зачем она нужна**, **как устроена** (формулы), **какие файлы/строки** её реализуют, **какие экраны** её показывают.
- Формулы приведены в псевдокоде (близко к TypeScript) и совпадают с фактическими вычислениями в `src/services/`.
- Ссылки на файлы идут в формате `путь:строка`, чтобы можно было сразу перейти к коду.

## Высокоуровневая карта систем

```
┌──────────────────────────── ИГРОВОЙ ЦИКЛ (неделя = 7 дней) ─────────────────────────┐
│                                                                                     │
│  модификаторы  →  клиенты  →  пропускная способность  →  выручка                    │
│       ▲              ▲                  ▲                    │                      │
│       │              │                  │                    ▼                      │
│   сезонность    репутация          сотрудники            −расходы                   │
│   реклама       качество           кассы                 −налоги                    │
│   события       бренд              синергии              −подписки                  │
│   NPC           лояльность         апгрейды              −pain losses               │
│                                                              │                      │
│                                                              ▼                      │
│                                                         прибыль недели              │
│                                                              │                      │
│          ┌───────────────────────────────────────────────────┤                      │
│          ▼                                                   ▼                      │
│   проверка банкротства                              обновление достижений,          │
│   / выгорания / репутации                           уровней, вех, онбординга        │
│                                                                                     │
└─────────────────────────────────────────────────────────────────────────────────────┘
```

Прибыль → баланс → возможность покупать сервисы → меньше pain losses + синергии → больше прибыли. Проигрыш приходит через отрицательный баланс 3+ недели, нулевую репутацию 4+ недели или нулевую энергию.
