# 05. NPC и отношения

Система персонажей — одна из «необъявленных» в старом CLAUDE.md, но реализованная в коде. Семь NPC, с которыми игрок взаимодействует через события; у каждого уровень отношений и память о решениях.

Стартовые отношения задаются в `src/services/npcManager.ts` при создании игры.

## 5.2. Структура стейта NPC

```ts
Npc {
  id: string
  name: string
  role: string
  relationshipLevel: number   // 0..100
  isRevealed: boolean         // показан ли в UI (первый контакт открывает)
  memory: Array<{ week, eventId, choiceId, note }>  // последние 10 выборов
}
```

Поле `memory` хранит последние 10 выборов игрока в событиях, связанных с этим NPC. Используется для ветвления цепочек событий (см. 02-events.md).

## 5.3. Изменение отношений

Происходит только через события. В опции события есть поле `npcRelationshipDelta`:

```
applyEventConsequence(...):
  если event.npcId и option.npcRelationshipDelta:
    npcs[npcId].relationshipLevel = clamp(0, 100, current + delta)
    npcs[npcId].isRevealed = true
    npcs[npcId].memory.push({ week, eventId, choiceId, note })
    если memory.length > 10: сдвиг (оставить последние 10)
```

Дельты бывают от ±5 до ±25 в зависимости от значимости выбора.

## 5.4. Пассивные еженедельные эффекты

**Функция:** `applyNPCPassiveEffects(state)` — вызывается в `processWeek()` после завершения дней.

```
если npcs.mikhail.relationshipLevel >= 75:
    state.temporaryCheckMod += 0.04   // временный бонус к чеку (эконом на закупках)

если npcs.svetlana.relationshipLevel >= 70:
    state.loyalty += 1

если npcs.marina.relationshipLevel >= 65:
    state.reputation += 1

если npcs.viktor.relationshipLevel >= 70:
    state.temporaryCheckMod += 0.02
```

Эти бонусы скромные, но накапливаются и сильно влияют на долгую игру.

## 5.5. Разблокировка опций в событиях

Некоторые варианты в событиях требуют определённого уровня отношений:

- `option.requiredNpcRelationship: { npcId, minLevel }` — вариант неактивен, если отношения ниже;
- пример: Виктор открывает «льготный кредит» при relationship ≥ 60;
- Анна при relationship ≥ 70 позволяет выбрать «не воевать, договориться» в ответе на её акции.

## 5.6. Примеры цепочек с NPC

- **inspector_chain** (Петров): выбор подкупить / пойти на проверку → через 2 недели следствие, результат зависит от уровня отношений с Петровым.
- **supply_collapse** (Михаил): если с поставщиком хорошие отношения — задержка 2 дня, если плохие — 5 дней и −30% товара.
- **rival_expansion** (Анна): Анна открывает филиал, при relationship ≥ 70 есть «заключить партнёрство», иначе «биться ценами».

## 5.7. Где видно NPC

- **Карточка события в DashboardView** и **EventModal** — портрет (emoji), имя, роль.
- **DecisionLogView** — у каждой записи виден `npcId`, можно фильтровать по персонажу.
- **StatisticsView** (косвенно) — расширенные цепочки и отношения влияют на итоговые числа.
- Выделенного «экрана NPC» в UI нет — отношения проявляются через контекст событий и пассивные бонусы.

## Ключевые файлы

- `src/constants/npcEvents.ts`, `src/constants/eventChains.ts` — события, меняющие отношения.
