# 03. Сервисы Контура, синергии, Pain Engine

Сервисы Контура — главный «контент‑крючок» игры. Они стоят денег (подписка в год), но дают прямые бонусы к экономике, блокируют часть событий и активируют синергии.

## 3.1. Семь сервисов

Конфигурация — в `src/constants/business.ts` (массив сервисов), тип — в `src/types/game.ts`.

| Сервис | Цена/год | Основной эффект |
|--------|----------|-----------------|
| **Контур.Маркет** | 48 000 ₽ | +20% пропускной, +15% средний чек, −20% списаний просрочки |
| **Контур.Банк** | 36 000 ₽ | Кредиты под 5%, −30% энергии на операции, 40% клиентов без Банка платят мимо кассы |
| **Контур.ОФД** | 12 000 ₽ | Онлайн-касса, защита от штрафов ФНС, синергия с Маркетом |
| **Контур.Диадок** | 24 000 ₽ | +5% клиентов, −25% энергии на закупки, защита от штрафов за бумагу |
| **Контур.Фокус** | 24 000 ₽ | +1 репутация/день, проверка поставщиков |
| **Контур.Эльба** | 36 000 ₽ | +2 лояльности/день, автоматическая бухгалтерия, −35% энергии |
| **Контур.Экстерн** | 48 000 ₽ | −2% к ставке налога, онлайн-отчётность |

Все бонусы применяются в `economyEngine.ts` и `weekCalculator.ts` как поля модификаторов (`capacityBonus`, `checkBonus`, `reputationBonus`, `loyaltyBonus`, `taxSaving`, `energySaving`).

Подписка списывается равномерно: `annualPrice / 365` каждый день.

## 3.2. Синергии

**Файл:** `src/services/synergyEngine.ts`, конфиг — `src/constants/business.ts:195-220`.

Когда одновременно активны определённые сервисы, добавляется дополнительный бонус.

Примеры реализованных синергий:

| Условие | Бонус |
|---------|-------|
| Market + OFD | +1 репутации/день (полный контроль выручки) |
| Market + Bank | −5% энергии, быстрые операции |
| OFD + Diadoc + Fokus | +2% клиентов (чистый документооборот) |
| OFD + Diadoc + Fokus + Elba | +1 лояльности/день |
| Все 7 | легендарный «бренд-эффект» в `qualityModifier.ts` (до +40% клиентов) |

Расчёт:

```
calculateSynergyModifiers(state):
  result = пустые бонусы
  для каждой синергии из SYNERGIES:
    если все требуемые сервисы активны:
      result.capacityBonus  += syn.capacityBonus
      result.checkBonus     += syn.checkBonus
      result.reputationBonus += syn.reputationBonus
      result.loyaltyBonus   += syn.loyaltyBonus
  return result
```

Результат добавляется к дневным модификаторам в `economyEngine.ts`. UI показывает активные синергии справа в `MainScreen` и на панели `ServicePanel`.

## 3.3. Pain Engine — потери без сервисов

**Файл:** `src/services/painEngine.ts`.

Идея: если сервис не подключён, игра каждый день/неделю отнимает у игрока деньги, чтобы мотивировать его оформить подписку.

| Сервис | Срабатывает | Потеря |
|--------|-------------|--------|
| **Bank** | каждый день | 40% клиентов платят мимо кассы: `revenue × (1 − 0.6)` |
| **Market** | каждый день | ошибки ручного учёта: `8% × totalCategoryRevenue` |
| **OFD** | 10% шанс в день | штраф ФНС: `15% × revenue` |
| **Diadoc** | каждый 10-й день | задержка поставок: `2% × revenue` |
| **Fokus** | каждый 17-й день | мошенник-поставщик: `5–10% × balance` |
| **Elba** | каждый 25-й день, только если profit > 0 | ошибка в декларации: `15% × profit` |
| **Extern** | каждый 31-й день | блокировка счёта: `2 × revenue` (потеря двухдневной выручки) |

Формула целиком:

```
calculatePainLosses(state, revenue, profit, totalCategoryRevenue):
  day = currentWeek * 7 + dayOfWeek
  losses = { bank: 0, market: 0, ofd: 0, diadoc: 0, fokus: 0, elba: 0, extern: 0 }

  // bank учитывается через bankPaymentRatio — отдельно в revenue

  if (!market) losses.market = round(totalCategoryRevenue * 0.08)
  if (!ofd  && Math.random() < 0.1) losses.ofd = round(revenue * 0.15)
  if (!diadoc && day % 10 === 0) losses.diadoc = round(revenue * 0.02)
  if (!fokus  && day % 17 === 0) losses.fokus = round(balance * (0.05 + rnd*0.05))
  if (!elba   && day % 25 === 0 && profit > 0) losses.elba = round(profit * 0.15)
  if (!extern && day % 31 === 0) losses.extern = round(revenue * 2)

  losses.total = sum(...)
  return losses
```

На практике у игрока без всех 7 сервисов каждый день незаметно съедает 30–50% потенциальной прибыли.

### Где видно потери

- **WeekResultsOverlay** показывает блок «Без сервисов Контура потеряно: X ₽» и топ‑3 источника потерь (работает в первые 6 недель, чтобы не отвлекать позже).
- **FinanceView** имеет секцию «Боль-потери» со списком по каждому неактивному сервису и пояснением.
- **OnboardingPanel** использует эти же формулировки, чтобы объяснить, зачем нужен очередной сервис.

## 3.4. Активация сервиса

`toggleService(id)` в `gameStore.ts`:

```
если включаем:
  списать первую дневную долю подписки
  services[id].active = true
  проверить, не разблокировались ли новые синергии
если выключаем:
  services[id].active = false
  пересобрать модификаторы
```

Сервис можно активировать также из варианта события с `activatesService`. В этом случае подписка «дарится», но дальнейшие дневные списания идут как обычно.

## Экраны, где это видно

- **DesktopKontur** (`src/components/design-system/DesktopKontur.tsx`) — главная витрина сервисов: 7 карточек 2×3 с ценой, эффектом, кнопкой «Подключить». Справа показывает «экономия от Контура» = сумма предотвращённых pain‑потерь.
- **ServicePanel** — компактный коллапсируемый список сервисов в правой панели desktop‑лейаута и в мобильной вкладке «Сервисы».
- **FinanceView** — блок подписок в расходах + список pain‑потерь.
- **Тосты MainScreen** — при активации сервиса показывается короткое объяснение эффекта, при выборе «варианта с Контуром» в событии — «сэкономлено X ₽».

## Ключевые файлы

- `src/services/painEngine.ts` — расчёт потерь без сервисов.
- `src/constants/business.ts` — цены, параметры, каталог синергий.
