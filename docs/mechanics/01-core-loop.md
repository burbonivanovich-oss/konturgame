# 01. Основной цикл и экономика

## Что это

Сердце игры — функция `processWeek()` в `src/services/weekCalculator.ts`. Игрок нажимает «Следующая неделя», и движок проигрывает 7 дней подряд, аккумулируя доход, расходы, события, боли и обновляя все глобальные показатели (репутация, лояльность, энергия, качество).

Экономическая математика живёт в `src/services/economyEngine.ts`. Она вычисляет, сколько клиентов пришло, сколько обслужено, какая выручка и сколько денег ушло на налоги/подписки/зарплаты.

## Недельный цикл

Псевдокод `processWeek()`:

```
инициализация: расчёт бонусов сотрудников, старт NPC-памяти

для каждого из 7 дней:
  1. собрать модификаторы (сезонность, реклама, события, бренд, конкурент)
  2. клиенты       = calculateClients(baseClients, modifiers)
  3. пропускная    = calculateCapacity(state) + employeeBonus * 10%
  4. обслужено     = min(клиенты, пропускная)
  5. упущено       = клиенты − обслужено
  6. выручка       = обслужено × средний_чек  (или сумма по категориям)
  7. штраф касс    = overflowPenalty(обслужено, throughput) × выручка
  8. налог         = выручка × (13% − taxSaving)
  9. расходы дня   = закупки + подписки/365 + коммуналка (250 ₽)
  10. pain loss    = calculatePainLosses(...)  (см. 03-services-and-pain.md)
  11. репутация    += −0.2 за каждого упущенного
  12. лояльность   += штраф за перегруз, бонус за хороший сервис
  13. обновить таймеры временных модификаторов

после 7 дней:
  - обновить качество (updateQualityWeekly)
  - списать энергию предпринимателя (базово 20 + цена управления персоналом)
  - проверить банкротство (баланс <0 три недели) / нулевую репутацию / выгорание
  - проверить условия победы (victoryChecker)
  - проверить вехи (week10/20/30)
  - выдать новые достижения (achievementChecker)
  - продвинуть онбординг (onboardingEngine)
  - применить пассивные эффекты NPC (npcManager)
  - запустить очередной шаг цепочек событий
  - сгенерировать 1–2 новых события (eventGenerator)
  - currentWeek += 1
```

Ежемесячные расходы (аренда + базовая зарплата) списываются раз в 4 недели — константа `MONTHLY_CYCLE_WEEKS` в `weekCalculator.ts:281`.

### Фазы недели в UI

В коде есть поле `weekPhase: 'summary' | 'actions' | 'events' | 'results'`. Жёстких блокировок по фазам нет (в GDD описан 4‑фазный цикл, в коде он мягкий): оверлеи `WeekSummaryOverlay` и `WeekResultsOverlay` открываются на фазах `summary` и `results`, между ними игрок свободно нажимает действия и решает события.

## Экономические формулы

### Клиенты в день

```
clients = round( baseClients
               × (1 + seasonalMod + advertisingMod + eventClientMod)
               × reputationMultiplier
               × qualityMultiplier
               × brandMultiplier )
```

- `baseClients` — из `BUSINESS_CONFIGS` (`src/constants/business.ts`): 15 / 18 / 6 (shop / cafe / beauty).
- `seasonalMod` — таблица сезонов в `business.ts`, диапазон от −15% (кафе зимой) до +22% (кафе летом).
- `advertisingMod` — результат стека активных кампаний. Повторяющиеся кампании получают убывающую отдачу: `CAMPAIGN_DIMINISHING_FACTORS = [1.0, 0.5, 0.3, 0.2]`.
- `reputationMultiplier` — от 0.5 (репутация ~0) до 1.1 (репутация ~100).
- `qualityMultiplier` — `qualityModifier.ts`: −30% при качестве <40, +20% при >70.
- `brandMultiplier` — «бренд‑эффект» из `qualityModifier.ts:43-78`: при высоких репутации + лояльности + качестве даёт до +40% клиентов.

### Пропускная способность

```
capacity = baseCapacity
         × (1 + (market.active ? 0.2 : 0)
              + (loyalty >  80 ?  0.1 : 0)
              + (loyalty <  30 ? −0.15 : 0)
              + upgradeCapacityBonus
              + synergyCapacityBonus)
         + employeeCapacityBonus  // до +10% на сотрудника
```

`baseCapacity`: shop 35, cafe 40, beauty‑salon 20 клиентов/день (см. `business.ts`).

### Средний чек

```
avgCheck = round( baseCheck
                × (1 + checkBonus
                     + advertisingCheckPenalty
                     + qualityPremium       // +15% при q>80, −15% при q<30
                     + brandCheckMod) )     // до +10% при легендарном бренде
```

Базовые чеки: 100 ₽ / 70 ₽ / 400 ₽.

### Выручка дня

Два пути:

- **Простой** (без ассортимента): `revenue = served × avgCheck`.
- **По категориям** (shop/cafe c включённым ассортиментом):
  `revenue = calculateCategoryRevenue(state) × (1 + synergyRevenueBonus + brandRevenueMod)`.
  Логика — в `assortmentEngine.ts` (см. 04-operations.md).

### Штраф за переполнение касс

`cashRegisterEngine.ts`:

```
overflow = served − totalRegisterThroughput
groups   = floor(overflow / 5)
penalty  = min(0.8, groups × 0.1) × revenue
```

То есть при 150 клиентах и кассах на 60 теряется до 80% дневной выручки.

### Налог

```
taxRate   = 0.13 − taxSaving       // сервис extern даёт −0.02
taxAmount = dayRevenue × taxRate
```

В реальности в коде это УСН 6% (см. FinanceView), но внутри движка поле называется налог и собирает все обязательные платежи.

### Расходы дня

- Закупки товара: если включены категории — их `dailyCost`.
- Подписки Контура: сумма `annualPrice` по активным сервисам, делённая на 365.
- Коммуналка: `DAILY_UTILITIES = 250 ₽`.
- Кассы: разово при покупке (`cashRegisterEngine.ts:45-53`), не ежедневно.
- Аренда и базовая зарплата: раз в 4 недели.
- Зарплата нанятых сотрудников: сумма `salary` каждого, делённая на 4 недели (`employeeManager.ts:getWeeklySalaryCost`).

### Кредиты

Описаны в типе `Loan` и в `FinanceView`:

| Тип | Сумма | Срок | Ставка |
|-----|-------|------|--------|
| Микрозайм (МФО) | 50 000 ₽ | 2 недели | 15% (и до 30% годовых без Контур.Банка) |
| Стандартный (Банк) | 100 000 ₽ | 4 недели | 10% |
| Долгосрочный (Банк) | 200 000 ₽ | 12 недель | 8% |

Контур.Банк удешевляет кредит и снижает операционную энергию на банковских действиях на −30%.

## Экраны, на которых видна эта механика

- **MainScreen → DashboardView**: 4 KPI‑карточки (баланс, прибыль, расходы, клиенты), виджет очереди (обслужено/упущено).
- **WeekResultsOverlay**: подробный разбор недели — выручка, закупки, налог, подписки, списание просрочки, pain‑потери.
- **FinanceView**: ежедневные расходы, месячные обязательства, прогноз на 4 и 8 недель, управление кредитами.
- **Indicators / Status bar**: репутация, лояльность, энергия, качество, обслуженность дня.
