# 06. Онбординг

Онбординг — система пошагового обучения, которая разблокирует сервисы и вкладки UI по мере прохождения. Без онбординга все 7 сервисов Контура были бы доступны сразу, игрок утонул бы в опциях — поэтому в коде реализован `SERVICE_UNLOCK_MAP`, жёстко запрещающий активировать сервисы, пока не пройден нужный этап.

## 6.1. Пять стадий

**Файлы:** `src/services/onboardingEngine.ts`, `src/constants/onboarding.ts`.

### Stage 0 — основы (дни 1–3)

Первые уроки, без сервисов. Шаги:

1. Приветствие.
2. «День — ваш ход».
3. Попробовать запустить день (wait-trigger: `first_day_completed`).
4. События и выборы.
5. Как читать финансы.
6. Энергия предпринимателя.
7. Что такое банкротство.
8. Ассортимент.
9. Готов к самостоятельности.

Доступные сервисы: `[]`.

### Stage 1 — приём оплаты (дни 4–14)

Боль: 40% клиентов не могут заплатить без терминала. Решение — **Контур.Банк**, касса, **Контур.ОФД**.

Доступные сервисы: `[bank, ofd]`. Условие перехода: день ≥ 15, bank и ofd активны, куплена хотя бы одна касса.

### Stage 2 — учёт товаров (дни 15–28)

Боль: 8% потерь от ручного учёта. Решение — **Контур.Маркет**.

Доступные сервисы: `[bank, ofd, market]`. Условие перехода: день ≥ 29, market активен.

### Stage 3 — документы и проверки (дни 29–42)

Боль: штрафы за документооборот и за сомнительных контрагентов. Решение — **Контур.Диадок** и **Контур.Фокус**.

Доступные сервисы: `[bank, ofd, market, diadoc, fokus]`. Условие перехода: день ≥ 43, diadoc активен.

### Stage 4 — отчётность и налоги (дни 43+)

Боль: ошибки деклараций, налоговые штрафы. Решение — **Контур.Эльба** и **Контур.Экстерн**.

Доступные сервисы: все 7. На дне ≥ 50 (при условии, что шаги просмотрены) — `onboardingCompleted = true`.

## 6.2. Служебные карты

`SERVICE_UNLOCK_MAP` (в `onboarding.ts`) задаёт, на какой стадии становится доступен каждый сервис. Проверяется в `toggleService()` — попытка включить недоступный сервис просто игнорируется, а UI подсвечивает иконку замком.

`VIEW_UNLOCK_MAP` аналогично разблокирует вкладки `KLeftRail`:

- `dashboard`, `ecosystem` — всегда;
- `warehouse` — с недели 2;
- `finance`, `operations` — всегда;
- `development` — с недели 2;
- `statistics` — с недели 7;
- `journal` — с недели 10.

При первой разблокировке вкладки в UI всплывает тост.

## 6.3. Типы шагов онбординга

В `onboardingEngine.ts` у каждого шага есть `type`:

- **intro** — просто информация, кнопка «Понял».
- **action** — требуется действие (например, нажать «Купить кассу»). Панель подсвечивает нужную вкладку и блокирует переход, пока действие не выполнено.
- **wait** — ждём игрового триггера.

### Триггеры ожидания

Проверяются в `checkOnboardingTriggers(state)`:

| Триггер | Условие |
|---------|---------|
| `first_day_completed` | `currentWeek > 1 || dayOfWeek > 0` |
| `first_event_shown` | `pendingEvent !== null` |
| `first_event_resolved` | `triggeredEventIds.length > 0` |
| `low_energy` | `entrepreneurEnergy < 50` |
| `negative_balance` | `balance < 0` |
| `low_stock` | `stock / capacity < 25%` |

При срабатывании триггера шаг автоматически помечается как пройденный.

## 6.4. Флоу продвижения

```
advanceOnboarding(state):
  stepIndex += 1
  если stepIndex >= шагов в стадии:
    если стадия <= 4 и условие перехода выполнено:
      stage += 1
      stepIndex = 0
    иначе:
      onboardingCompleted = true

processWeek() в конце недели вызывает advanceOnboarding,
но только если текущий шаг пройден (wait-триггер сработал
или action завершён).
```

## 6.5. Связь с pain engine

Онбординг и pain engine работают вместе: стадия выводит конкретную боль, а pain engine уже списывает с игрока деньги за отсутствие сервиса. Таким образом игрок чувствует проблему на своих финансах и сам заинтересован нажать «Подключить» в конце стадии.

## Экраны

- **OnboardingPanel** (`src/components/OnboardingPanel.tsx`) — основной UI:
  - коллапсируемая панель сверху, в десктопе над контентом, в мобиле — на вкладке «Сервисы»;
  - индикатор прогресса по стадиям (5 точек);
  - текст текущего шага;
  - кнопки «Понял», «Открыть вкладку X», «Подключить Y» в зависимости от типа шага;
  - для `wait`-шагов показывается пульсирующий индикатор «ждём действия».
- **ServicePanel / DesktopKontur** — иконки закрытых сервисов помечены замком с подсказкой «Откроется на этапе X».
- **KLeftRail** — заблокированные вкладки серые с замком.

## Ключевые файлы

- `src/services/onboardingEngine.ts` — логика шагов, триггеров, продвижения.
- `src/constants/onboarding.ts` — определения стадий, шагов, `SERVICE_UNLOCK_MAP`, `VIEW_UNLOCK_MAP`.
- `src/components/OnboardingPanel.tsx` — UI панели онбординга.
