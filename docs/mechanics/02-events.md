# 02. События, цепочки, моральные дилеммы, новости города

В игре есть несколько независимых слоёв «событийки». Все они реализованы в `src/services/eventGenerator.ts` и связанных константах.

## 2.1. Обычные события

**Источник:** `src/constants/` (базовый каталог событий встроен в генератор) + `src/services/eventGenerator.ts`.

Событие — это карточка с описанием и 2–3 вариантами ответа. Каждый вариант задаёт последствия: деньги, репутация, лояльность, энергия, временные модификаторы клиентов/чека на N дней, отношения с NPC, активация сервиса, триггер следующей цепочки.

### Алгоритм генерации

```
generateEvent(day, state):
  1. отфильтровать кандидатов:
     - oneTime-события, которые уже срабатывали → пропустить
     - не попал в окно [dayMin..dayMax]
     - репутация не в нужном диапазоне
     - не выполнены условия по активным/отсутствующим сервисам
  2. бросить кубик на randomChance (обычно 4–8% в день)
  3. выбрать случайного кандидата
  4. вернуть Event с массивом option’ов
```

В итоге за неделю игрок получает в среднем 1–2 события — `processWeek()` вызывает генератор в конце недели.

### Структура последствий

Поля option’а, применяемые в `applyEventConsequence`:

- `balanceDelta` — разовая сумма к балансу;
- `reputationDelta`, `loyaltyDelta`, `energyDelta` — изменения основных показателей;
- `clientModifier { value, days }`, `checkModifier { value, days }` — временные модификаторы;
- `activatesService` — включает сервис Контура (подписка оплачена «бонусом»);
- `npcRelationshipDelta` — изменение отношений с NPC (см. 05-npc-system.md);
- `chainFollowUpId` — идентификатор следующего звена цепочки с отложенным триггером (обычно +2 недели).

### Примеры базовых событий

| ID | Событие | Без сервиса | С сервисом Контура |
|----|---------|-------------|--------------------|
| `TAX01` | Налоговая проверка | −80 000 ₽ | `extern` → −5 000 ₽ |
| `SUPPLY01` | Срыв поставки | −35 000 ₽ и −30% клиентов на неделю | `market` → −2 000 ₽ |
| `OFD_FINE01` | Штраф без онлайн-кассы | −60 000 ₽ | — |
| `DIADOC01` | Штраф за бумажный документооборот | −30 000 ₽ | `diadoc` → 0 ₽ |
| `MARKET_STOCKOUT` | Дефицит товара | −45 000 ₽ | — |

Варианты выбора, в которых есть «альтернатива с Контуром», помечены `isContourOption: true`. Для них MainScreen считает «сэкономлено X ₽» и показывает тост.

## 2.2. Моральные дилеммы

**Файл:** `src/constants/moralDilemmas.ts`.

Отдельный каталог событий с флагом `isMoralDilemma: true`. Предлагает выбор между прибылью и этикой (например: обмануть налоговую — получить деньги, но потерять репутацию). Механика такая же, как у обычных событий, но у них жёсткий баланс: один вариант всегда «хорошо для денег, плохо для репутации», другой — наоборот.

## 2.3. NPC‑события

**Файл:** `src/constants/npcEvents.ts` + `src/services/npcManager.ts`.

Событие с заполненным `npcId` привязано к конкретному персонажу (см. 05-npc-system.md). Выбор меняет не только деньги/репутацию, но и `relationshipLevel` с этим NPC. Варианты, которые становятся доступны, могут зависеть от текущего уровня отношений. Решение попадает в «память» NPC (последние 10 выборов), что используется в цепочках.

## 2.4. Цепочки событий

**Файл:** `src/constants/eventChains.ts`.

Многонедельные сюжетные линии: выбор в первом звене → триггер через 2 недели → новое событие уже с контекстом предыдущего выбора.

Примеры цепочек:

- `inspector_chain` — приходит инспектор Петров, через 2 недели выставляет следствие выбора;
- `legacy_chain` — открывается при репутации ≥ 70: старые долги, ответственность за район;
- `rival_expansion` — конкурент расширяется, требуется ответ;
- `supply_collapse` — поставщик срывает сроки, нужно стабилизировать поставки.

Алгоритм:

```
1. на неделе X генератор запускает стартовое событие цепочки
2. при выборе option: записать chainFollowUpId и пометить триггер на неделю X+2
3. на неделе X+2 автоматически подставляется follow-up
4. варианты follow-up могут зависеть от NPC-отношений и предыдущего выбора
```

## 2.5. Газета города

**Файл:** `src/constants/cityNewspaper.ts`.

Раз в 10 недель случается «новость района» — обще‑игровое событие, которое влияет на весь бизнес: фестиваль увеличивает клиентов у кафе, закрытие улицы снижает трафик у магазина и т. д. Функция — `getNewspaperForWeek(week)`.

## 2.6. Постоянные клиенты

**Файл:** `src/constants/recurringCustomers.ts`.

Для каждого типа бизнеса есть свой «фирменный» постоянный клиент:

- Магазин — Николай Петрович;
- Кафе — Катя;
- Салон красоты — Мария Ивановна.

Механика:

- клиент «появляется», когда лояльность ≥ 40%;
- пока он ходит, каждое появление даёт +1 лояльность;
- если в неделю его не было (низкий сервис / плохое качество / плохие выборы) — он «пропадает», даёт −1 лояльность и напоминание в `WeekResultsOverlay`.

## 2.7. Микрособытия

**Файл:** `src/constants/dailyMicroEvents.ts` + `src/services/microEventGenerator.ts` (логика) + `src/components/modals/MicroEventModal.tsx`.

Слой мелких ежедневных событий без развилок — просто «информация + небольшое изменение показателя»:

- «Клиент оставил отзыв» → +1 лояльность;
- «Обнаружена ошибка в расходах» → небольшие минусы;
- «Заметка в соцсетях» → ±1 репутация.

Они генерируются отдельно от основных событий и показывают отдельную модалку, чтобы не перекрывать «большую» карточку события.

## Как применяются последствия

`applyEventConsequence(state, event, optionId)` в `eventGenerator.ts`:

```
state.balance         += option.balanceDelta
state.reputation       += option.reputationDelta
state.loyalty          += option.loyaltyDelta
state.entrepreneurEnergy += option.energyDelta

если option.clientModifier: state.temporaryClientMods.push({...})
если option.checkModifier:  state.temporaryCheckMods.push({...})
если option.activatesService: state.services[id].active = true
если option.npcRelationshipDelta: npcs[npcId].relationshipLevel += delta
если option.chainFollowUpId:
   state.pendingChainEvents.push({id, triggerWeek: currentWeek + 2})

state.triggeredEventIds.push(event.id)
state.decisionLog.push({week, type, text, impact, npcId})
```

Временные модификаторы уменьшаются в `processWeek()` — поле `days` уменьшается каждый день, модификатор удаляется на нуле.

## Экраны, на которых видна механика

- **MainScreen / DashboardView**: главная карточка `pendingEvent` с портретом NPC и вариантами.
- **EventModal** (мобильный): то же самое в формате модалки.
- **MicroEventModal**: мелкие ежедневные события.
- **DecisionLogView** (`journal` tab): хронология всех принятых решений с цветовой кодировкой (положительное / отрицательное / нейтральное) и типом (Решение / Цепочка / NPC / Газета / Клиент / Веха).

- `src/services/eventGenerator.ts` — генерация, фильтры, применение последствий.

- `src/constants/eventChains.ts`, `src/constants/moralDilemmas.ts`, `src/constants/npcEvents.ts`, `src/constants/cityNewspaper.ts`, `src/constants/recurringCustomers.ts`, `src/constants/dailyMicroEvents.ts` — каталоги событий по слоям.
- `src/components/modals/EventModal.tsx`, `MicroEventModal.tsx`, `src/components/views/DecisionLogView.tsx` — UI.
