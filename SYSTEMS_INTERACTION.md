## 🔄 Основные циклы взаимодействия

### **Цикл 1: Доход → Расходы → Баланс → Действия**

```
Дневной доход
    │
    ├─→ Если баланс > 50,000₽ → Можно купить улучшение
    │
    ├─→ Если баланс > 30,000₽ → Можно запустить рекламу
    │
    └─→ Если баланс < 10,000₽ → АВАРИЙНОЕ СОСТОЯНИЕ
         │
         └─→ Нужно срочно сократить расходы или увеличить доход
```

**Пример (Магазин, неделя 2):**
```
День 10: +4,500₽ доход - 500₽ коммун - 300₽ касса = +3,700₽
Баланс: 100,000₽ - 25,000₽ (аренда) - 15,000₽ (зарплата) + 35,700₽ = 95,700₽

День 30: Конец месяца (аренда + зарплата снова)
Баланс: 95,700₽ - 25,000₽ - 15,000₽ + 37,000₽ = 92,700₽

Вывод: После 2-го месяца баланс в минусе если не было прибыли!
Решение: Нужны сервисы (Маркет +15% чек) или реклама (+25% клиентов)
```

### **Цикл 2: Репутация → Клиенты → Чек → Прибыль → Репутация**

```
Высокая репутация (80+)
    │
    ├─→ +1.1 множитель клиентов
    │
    ├─→ Больше потенциальных клиентов
    │
    ├─→ Если пропускная хватает → больше обслужено
    │
    ├─→ Больше выручка
    │
    └─→ Больше денег на сервисы
         │
         └─→ Сервисы добавляют эффекты репутации
             │
             └─→ ещё выше репутация (положительный цикл)
```

**Обратный цикл (ловушка):**
```
Низкая репутация (< 20)
    │
    └─→ ×0.5 множитель клиентов
         │
         └─→ Мало клиентов → мало выручки
              │
              └─→ Нет денег на сервисы
                  │
                  └─→ Сервисы бы помогли поднять репутацию
                      │
                      └─→ но денег нет → ещё ниже репутация
                         │
                         └─→ ЛОВУШКА БЕДНОСТИ
```

**Как избежать:** Подключить Fokus или Market+OFD синергию (пассивно +1-2 репутация/день)

### **Цикл 3: Лояльность → Пропускная → Обслужено → Лояльность**

```
Высокая лояльность (> 80)
    │
    ├─→ +10% пропускная способность
    │
    ├─→ Больше клиентов можно обслужить
    │
    ├─→ Больше выручка
    │
    └─→ Могу купить улучшение (Эльба) → +2 лояльности/день
         │
         └─→ ещё выше лояльность (положительный цикл)
```

**Обратный цикл (ловушка):**
```
Низкая лояльность (< 30)
    │
    └─→ -15% пропускная способность
         │
         └─→ Меньше клиентов обслужено
              │
              └─→ Потеряны клиенты → -10 лояльности за переполнение
                  │
                  └─→ ещё ниже лояльность
                     │
                     └─→ ЛОВУШКА НЕВЫПОЛНЕНИЯ
```

**Как избежать:** Подключить Эльба (+2 лояльности/день пассивно)

### **Цикл 4: Сервисы → Синергии → Эффекты → Победа**

```
Подключить Маркет (день 5)
    │
    ├─→ +20% пропускная, +15% чек, -20% потери товара
    │
    └─→ Доход растёт на 35% примерно

Подключить ОФД (день 10)
    │
    ├─→ Синергия Market+OFD активируется
    │
    └─→ +2 репутация/день пассивно
         │
         └─→ Клиентов больше

Подключить Эльба (день 15)
    │
    ├─→ +2 лояльности/день пассивно
    │
    ├─→ Синергия Bank+Elba (если Bank активен)
    │
    └─→ +5% выручка
         │
         └─→ МОЩНЫЕ СИНЕРГИИ РАБОТАЮТ!
```

## ⚙️ Взаимодействие систем по пункта

### **1. Экономическая система ↔ Сервисы Контура**

| Сервис | Эффект на доход | Эффект на расход | Решаемая проблема |
|--------|---|---|---|
| **Market** | +35% (чек + пропускная) | +2000₽/мес | Потери товара, низкий чек |
| **Bank** | 0% прямой | -30% энергии (экономит деньги) | Дефицит капитала |
| **OFD** | +2 репутация/день (косвенно) | +500₽/мес | Отсутствие кассы |
| **Diadoc** | +5% клиентов | +1000₽/мес | Медленные поставки |
| **Fokus** | +1 репутация/день (косвенно) | +1000₽/мес | Недобросовестные поставщики |
| **Elba** | +2 лояльности/день (косвенно) | +1500₽/мес | Сложность управления |
| **Extern** | -2% налогов (прямой) | +2000₽/мес | Высокие налоги |

**Вывод:** Каждый сервис решает реальную бизнес-проблему и требует инвестиций.

### **2. События ↔ Сервисы**

События дизайнированы чтобы показать **ценность сервисов**:

| Событие | Без сервиса | С сервисом | Сэкономлено |
|---------|---|---|---|
| Налоговая проверка | -30,000₽ | -5,000₽ (Extern) | 25,000₽ |
| Проблема с поставщиком | -5,000₽ | -2,000₽ (Market) | 3,000₽ |
| Недовольство персонала | -10,000₽ | -5,000₽ (Elba) | 5,000₽ |
| Хищение товара | -8,000₽ | -2,000₽ (CCTV upgrade) | 6,000₽ |

**Игровой смысл:** События вынуждают игрока задуматься о сервисах.

### **3. Товар ↔ Репутация**

Просрочка товара → потери денег → нет денег на сервисы → ниже репутация

```
Без Market:
  День 10 (просрочка): -80% стоимости товара = -192₽
  × 12 раз в месяц = -2304₽ потерь
  
С Market:
  День 10: -64% стоимости = -153₽
  × 12 раз = -1844₽
  Сэкономлено: 460₽/месяц
  
За год: 460₽ × 12 = 5520₽ сэкономлено
```

### **4. Энергия ↔ Все операции**

Энергия ограничивает количество действий в неделю:

```
100 энергии / неделя

Без сервисов:
  - Закупка (-25) × 1 раз/неделя
  - Реклама (-20) × 1 раз/неделя  
  - Изменение ассортимента (-15) × 1 раз
  Итого: 60 энергии на 3 операции

С Elba (-35% энергии на операции):
  - Закупка (-16) × 1 раз
  - Реклама (-13) × 1 раз
  - Изменение ассортимента (-10) × 1 раз
  Итого: 39 энергии → осталось 61 на другое
```

**Вывод:** Elba позволяет делать больше операций за неделю.

### **5. Уровни ↔ Онбординг**

Открытие механик по уровням создаёт прогрессию в сложности:

```
Уровень 1-2: Базовый доход/расход
Уровень 3-4: Видны рекламы, синергии
Уровень 5-6: Открыты все сервисы, виден налог
Уровень 7-8: Кассовые аппараты, ассортимент
Уровень 9-10: Спецпредложения, доступна победа
```

## 🎯 Баланс между системами

### **Золотой баланс (день 30, месяц 1)**

При правильном выборе стратегии баланс должен быть:

```
Еженедельный доход:      +42,000₽ (6,000₽/день)
Еженедельные расходы:    -35,000₽ (5,000₽/день)
Еженедельная прибыль:    +7,000₽

Параметры:
- Репутация: 50+ (нормально, может быть выше)
- Лояльность: 70+ (хорошо, сервисы помогают)
- Баланс: ~80,000₽ (снизился на 20k после месячных платежей)
- Энергия: 100 (восстановилась на неделе)
```

### **Зоны опасности**

**Зона 1: Недостаточная прибыль**
```
Признаки:
- Еженедельная прибыль < 3,000₽ на неделю 2
- Баланс упал ниже 70,000₽ после месячных платежей
Решение: Срочно подключить сервис (Market) или рекламу
```

**Зона 2: Низкая репутация**
```
Признаки:
- Репутация < 30 на неделю 3
- Клиентов мало (множитель 0.7)
Решение: Подключить Fokus (+1/день) или Market+OFD (+2/день)
```

**Зона 3: Низкая лояльность**
```
Признаки:
- Лояльность < 40 на неделю 2
- 3+ дня переполнения касс
Решение: Подключить Elba (+2/день) или купить расширение зала
```

**Зона 4: Недостаток товара**
```
Признаки:
- Товар кончается до конца дня (stockouts)
- Потери клиентов по товару
Решение: Купить Market (FIFO управление) или закупить больше
```

## 🚨 Критические точки (Момент истины)

### **Критическая точка 1: День 10 (выбор первого сервиса)**

```
Вопрос: Какой сервис подключить первым?

Option A: Market (рекомендуется)
├─ Стоимость: +2000₽/мес = 7500₽ за месяц
├─ Эффект: +20% пропускная, +15% чек = +35% доход примерно
├─ ROI: 1 месяц окупается, потом профит
└─ Результат: ✓ УСПЕХ

Option B: Полный Kontour (все 7)
├─ Стоимость: +9500₽/мес = 31500₽ за месяц
├─ Эффект: +15% выручка, все синергии
├─ ROI: 2-3 месяца, опасно
└─ Результат: ✗ КРАХ (нет денег на аренду)

Option C: Ничего не покупать
├─ Стоимость: 0₽
├─ Эффект: нет улучшений
├─ Проблема: медленный рост, потери товара
└─ Результат: ⚠️ СКУЧНО
```

**Ошибка в выборе здесь ≈ 50% шанс поражения**

### **Критическая точка 2: День 30 (первый месячный платёж)**

```
Баланс перед платежом: должен быть > 55000₽
Платеж: -40000₽ (аренда + зарплата)
Баланс после: должен остаться > 15000₽ (подушка)

Если баланс < 55000₽:
├─ Решение 1: Был бы большой долг? Нет, Game Over
├─ Решение 2: Продать улучшения? Нельзя
└─ Решение 3: Запустить рекламу? Поздновато
```

**Ошибка в первый месяц ≈ 30% шанс поражения**

### **Критическая точка 3: День 60 (вторая синергия)**

```
На этот момент должно быть:
├─ Баланс: 100k+ (восстановился)
├─ Маркет: активен (день 5)
├─ ОФД: активен (день 10)
├─ Новый сервис: Elba или Diadoc или Fokus

Если нет нового сервиса к дню 60:
├─ Репутация может быть низкой (нет +1/день)
├─ Лояльность может быть низкой (нет +2/день)
├─ Скучно, нет мотивации
└─ Риск: игрок забросит
```

**Ошибка или скука здесь ≈ 20% шанс поражения**

## 🎮 Механики как линзы для управления сложностью

### **Как разработчик управляет сложностью**

1. **События** — добавляют неопределённость и необходимость выбора
2. **Уровни** — открывают механики постепенно
3. **Достижения** — показывают возможные стратегии
4. **Энергия** — замедляет игру, даёт паузу на раздумья
5. **Сервисы** — предлагают решения проблем
6. **Синергии** — вознаграждают планирование

### **Если игра слишком сложна на день 10:**
- Усилить начальный капитал (100k → 120k)
- Сделать Маркет дешевле (2000₽ → 1500₽)
- Добавить промокод на скидку первого месяца
- Усилить онбординг с подсказками

### **Если игра слишком скучна на день 30:**
- Добавить больше событий (3% → 5%)
- Усилить эффекты сервисов (синергии яснее видны)
- Добавить достижения за милестоны
- Показать прогресс к победе (сколько дней осталось)

- [ ] Баланс восстановился

- [ ] Третий сервис выбран (Elba или Diadoc)
- [ ] Лояльность растёт
- [ ] Уровень 5-6 достигнут
- [ ] 3-4 достижения получены

### **День 61-365: Оптимизация**
- [ ] Вся система в балансе
- [ ] Выполнены 2-3 из 4 условий победы
- [ ] Игрок ясно видит путь к победе
- [ ] Финальные синергии спланированы

## 🎯 Выводы

**Все системы игры спроектированы чтобы:**
1. **Поддерживать друг друга** — события решаются сервисами, сервисы улучшают параметры
2. **Создавать выбор** — нет одного "правильного пути", есть несколько стратегий
3. **Обучать игрока** — каждая механика показывает реальный бизнес-принцип
4. **Управлять сложностью** — от простой выживаемости к сложной оптимизации
5. **Мотивировать** — постоянное ощущение прогресса и новых возможностей

**Баланс достигается когда:**
- Игрок может выжить с базовой стратегией
- Но получит больше выгоды из оптимальной стратегии
- И будет гордиться своими решениями в конце
