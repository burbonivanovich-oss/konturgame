### **4. DIFFICULTY_AND_PLAYER_OPTIONS.md** — Сложность и стратегии
- **Объём:** ~3,000 слов
- **Фокус:** Как игрок воспринимает сложность и какие есть варианты
- **Аудитория:** Дизайнеры, тестировщики, гемеры

**Содержит:**
- Кривая сложности по дням (6 этапов)
- Подробное описание каждого этапа:
  - День 1-7: Выживание (80-90% сложность)
  - День 8-21: Выбор (60-70% сложность)
  - День 22-45: Синергия (50-60% сложность)
  - День 46-180: Оптимизация (30-40% сложность)
  - День 181-330: Стабильность (35-50% сложность)
  - День 331-365: Финиш (70-80% сложность)
- Матрица 9 основных стратегий:
  - A: Интересная консервативная (РЕКОМЕНДУЕТСЯ новичкам)
  - B: Сбалансированная стандарт
  - C: Агрессивная оптимальная
  - D, E, F: Специальные стратегии
  - G, H, I: Нишевые подходы
- 4 уровня сложности (CASUAL, NORMAL, HARD, INSANE)
- Таблица метрик успеха по неделям
- Индикаторы успеха по дням
- Чек-лист опыта игрока
- Рекомендуемый опыт для новичка
- Детальный walkthrough первого прохождения

**Ключевые стратегии:**
| Стратегия | Прибыль | Риск | Для кого |
|-----------|---------|------|----------|
| A (Интересная консервативная) | 10-15k/неделя | Низкий | Новичков |
| B (Сбалансированная) | 15-20k/неделя | Средний | Стандартных игроков |
| C (Агрессивная) | 25-35k/неделя | Высокий | Опытных |

## 🎯 Как использовать документацию

### **Для разработчика:**
1. Начните с **GAME_MECHANICS.md** — поймите как работают все числа
2. Переходите к **SYSTEMS_INTERACTION.md** — смотрите как они связаны
3. Используйте как справочник при работе с кодом
4. Проверяйте цифры в `src/constants/business.ts`

### **Для дизайнера:**
1. Начните с **GAME_DESIGN_DOCUMENT.md** — поймите видение
2. Дополните **DIFFICULTY_AND_PLAYER_OPTIONS.md** — смотрите как игрок это воспринимает
3. Используйте для баланса: если игра скучна (день 30) — добавьте событий
4. Смотрите стратегии — разработчик может выбрать любую, значит есть выбор

### **Для тестировщика:**
1. Начните с **DIFFICULTY_AND_PLAYER_OPTIONS.md** — знайте что искать
2. Используйте чек-листы по этапам — проверяйте критические точки
3. Смотрите критические параметры (баланс, репутация, лояльность)
4. Проводите тесты по 9 стратегиям — каждая должна быть выполнима

### **Для аналитика:**
1. **SYSTEMS_INTERACTION.md** — диаграммы и таблицы влияния
2. **DIFFICULTY_AND_PLAYER_OPTIONS.md** — метрики по неделям
3. Следите за удержанием: день 15, 30, 60, 180
4. Трекьте, какую стратегию выбирают игроки

### **Для маркетолога / Продакт-менеджера:**
1. **GAME_DESIGN_DOCUMENT.md** — уникальное предложение (EdTech + Kontour)
2. **DIFFICULTY_AND_PLAYER_OPTIONS.md** — время на первое прохождение (4-5 часов)
3. Основное СТП: "Управляй бизнесом и учись работать с Контуром"
4. Ключевой момент: день 10 (первый сервис) — вот где начинается веселье

## 🎓 Образовательная ценность

**Механики учат:**
1. **Экономика** — доход, расходы, налоги, прибыль, баланс
2. **Управление ресурсами** — деньги, время, товар, энергия
3. **Стратегическое планирование** — выбор инвестиций, синергии
4. **Риск-менеджмент** — критические точки, подушки безопасности
5. **Принятие решений** — события как дилеммы выбора
6. **Метрики бизнеса** — репутация, лояльность, прибыль
7. **Цифровизация** — сервисы Контура как решения проблем

**Контур.Маркет показывает:** Автоматизация = больше пропускная + меньше потери  
**Контур.Банк показывает:** Финансирование = возможность развития  
**Контур.ОФД показывает:** Кассы = бухучёт и документация  
**Контур.Диадок показывает:** ЭДО = ускорение поставок  
**Контур.Фокус показывает:** Проверка контрагентов = риск-менеджмент  
**Контур.Эльба показывает:** Бухгалтерия = управление персоналом  
**Контур.Экстерн показывает:** Отчётность = налоговая оптимизация
