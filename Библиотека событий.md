# **Библиотека событий: «Бизнес с Контуром»**

**Версия:** 1.0  
**Назначение:** описание всех ситуаций, которые требуют выбора игрока, с их триггерами, вариантами действий и последствиями.  
**Формат:** CSV / JSON для импорта в движок.

## **1. Структура одной записи (поля)**


| Поле | Тип | Описание |
| --- | --- | --- |
| id | string | Уникальный код события, например TAX_01 |
| title | string | Короткое название (для логов) |
| description | string | Текст, который видит игрок |
| trigger | string | Условие наступления (см. раздел 2) |
| business_type | string | Магазин / Кофейня / Салон / any |
| min_day | int | Минимальный день, с которого может случиться |
| max_day | int | Максимальный день (если не указано — без ограничения) |
| min_reputation | int | Минимальная репутация (если требуется) |
| max_reputation | int | Максимальная репутация |
| required_service | string | ID сервиса Контура, который должен быть активен для появления события (опционально) |
| option_1_text | string | Текст кнопки первого варианта |
| option_1_effect | string | Изменения ресурсов (см. синтаксис) |
| option_1_required | string | Если вариант требует сервис — указать ID |
| option_2_text | string | Текст кнопки второго варианта |
| option_2_effect | string | Изменения ресурсов |
| option_2_required | string | (опционально) |
| option_3_text | string | Текст кнопки третьего варианта (обычно с Контуром) |
| option_3_effect | string | Изменения ресурсов |
| option_3_required | string | (опционально) |
| is_contour_option | int | 1 если третий вариант от Контура (визуальное выделение) |

**Синтаксис эффектов:**  
`money:+5000` — добавить 5000 денег  
`money:-30000` — отнять 30000  
`reputation:+10` — увеличить репутацию на 10  
`reputation:-5` — уменьшить  
`loyalty:+15` — изменить лояльность персонала  
`stock:+50` — добавить товар на склад (с указанием срока годности?) — для простоты пока без срока, но можно добавить параметр `days_to_expire:10`  
`event:disable_week` — отключить другие события на 7 дней (спецэффект)  
`service:Маркет` — подключить сервис (если ещё нет) на стандартный срок

## **2. Триггеры событий**

Событие может наступить, если выполнены все условия `trigger`. Формат: логическое выражение с запятыми как AND.

Примеры триггеров:

- `day>5, day<20` — между 6 и 19 днём

- `reputation<30` — репутация ниже 30

- `has_service:ОФД` — активен сервис ОФД

- `business:Кофейня` — только для кофейни

- `random:0.05` — 5% шанс в день (если все остальные условия выполнены)

В коде можно комбинировать: `day>10, reputation<50, random:0.1`

## **3. Категории событий (для организации)**

1. **Налоговые и проверки** (штрафы, блокировки)

1. **Поставщики и склад** (просрочка, брак, мошенники)

1. **Персонал** (увольнения, переработки, конфликты)

1. **Клиенты и репутация** (жалобы, отзывы, конфликты в очереди)

1. **Чрезвычайные ситуации** (поломка кассы, пожар, кража)

1. **Маркетинговые возможности** (внезапная реклама, партнёрство)

1. **Сервисы Контура** (сбои, обновления, напоминания)

## **5. Примечания по реализации**

- События не должны наступать каждый день. Рекомендуемый максимум — 1 событие в 2 дня, в среднем 2–3 события в неделю.

- Приоритет триггеров: сначала проверяются события с явными требованиями (день, репутация, бизнес), затем случайные.

- Если одновременно подходят несколько событий — выбирается одно случайное из пула.

- Варианты с Контуром должны быть заметно выгоднее, но не бесплатными. Игрок должен видеть, что подписка окупается.

- Для каждого события можно добавить уникальную иконку или анимацию (позже, на этапе дизайна).
